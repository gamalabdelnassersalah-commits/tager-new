# Tager Clean MVP V2

نسخة تجريبية مستقلة للرفع على GitHub ثم Vercel.

## أهم التطويرات

- صور منتجات SVG واضحة لا تفقد الجودة عند الرفع.
- رفع صور المورد والمنتج مع تحقق من النوع والحجم والدقة (600×600 على الأقل).
- مناطق توصيل متعددة لكل مورد: محافظة، مركز، قسم/حي، رسوم الشحن، ومدة التوصيل.
- البحث عن المنتجات والموردين يعرض فقط المحافظات والمراكز والأقسام التي يخدمها الموردون فعليًا.
- صفحة متجر مستقلة لكل مورد تعرض منتجاته ومناطق التوصيل الخاصة به.
- حساب مالي للمورد يعرض:
  - إجمالي المبيعات.
  - نسبة وقيمة عمولة المنصة.
  - المبلغ المدفوع.
  - المبلغ المتبقي.
  - صافي المورد.
  - سجل الدفعات وطريقة الدفع ورقم المرجع.
- صفحة مالية للإدارة تعرض أرصدة جميع الموردين واعتماد أو رفض دفعات العمولة.
- رسوم السلة المميزة وعمولة كل مورد قابلة للتعديل.
- حساب الشحن في إتمام الطلب حسب منطقة كل مورد.

## حسابات التجربة

- الإدارة: `01000000000` — `Admin@123`
- المورد: `01111111111` — `Vendor@123`
- العميل: `01222222222` — `Customer@123`

## الرفع على Vercel

1. فك ضغط الملف.
2. ارفع الملفات الموجودة داخل المجلد إلى مستودع GitHub.
3. في Vercel اختر المستودع.
4. Framework Preset: `Other`.
5. اترك Build Command وOutput Directory فارغين.
6. اضغط Deploy.

## تنبيه مهم

هذه نسخة اختبار تحفظ البيانات داخل Local Storage في المتصفح نفسه. لكي يرى جميع المستخدمين نفس البيانات من أجهزة مختلفة، يجب ربط المرحلة التالية بقاعدة بيانات Supabase وتخزين صور فعلي.
