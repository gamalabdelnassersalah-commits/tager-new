# Tager Platform

منصة تجارة وتوريد متكاملة باللغة العربية لإدارة السوق من أول تسجيل المورد والمشتري حتى الطلب، التوصيل، الفاتورة، التسوية، الدعم، والتقارير.

## التشغيل

افتح ملف `index.html` مباشرة، أو ارفع مجلد `public` على Vercel / Netlify / أي استضافة ثابتة.

عند أول تشغيل تظهر صفحة إعداد حساب الإدارة. بعد إنشاء حساب الإدارة يتم الدخول إلى لوحة الإدارة واستكمال بيانات التشغيل الحقيقية.

## محتويات النظام

- صفحة رئيسية بتصميم فاتح راقي وهوية Tager.
- سوق منتجات وفئات وموردين وصفحة مستقلة لكل مورد.
- تسجيل مشتري وتسجيل مورد وتسجيل دخول واستعادة كلمة المرور.
- تتبع الطلب برقم الطلب ورقم الهاتف.
- سلة وإتمام طلب مع فحص تغطية المورد والمحافظة والمركز.
- دعم كود الخصم، التوصيل، رسوم السلة المميزة، والضريبة من داخل الإعدادات.
- لوحة مشتري لمتابعة الطلبات والدعم.
- لوحة مورد للبيانات، مناطق التغطية، المنتجات، الطلبات، الحساب المالي، والمستندات.
- لوحة إدارة للمستخدمين، المشترين، الموردين، المنتجات، الفئات، الطلبات، الاعتمادات، التشغيل، المخزون، الفواتير، المرتجعات، المالية، التسويات، التقارير، العروض، التنبيهات، المستندات، الدعم، المحتوى، الإعدادات، والبيانات.
- مركز تشغيل لمتابعة مسار الطلبات وتنبيهات المخزون والتغطية والطلبات الجديدة.
- مركز اعتمادات للموردين والمنتجات قبل الظهور في السوق.
- إدارة مخزون مع حد تنبيه منخفض وإمكانية تعديل الرصيد مع سجل حركة.
- إصدار فواتير من الطلبات المسلمة وطباعتها.
- تسجيل مرتجعات وربطها بالطلبات المسلمة.
- مركز مالي لحساب المبيعات، العمولات، الرسوم، المدفوعات، ومستحقات الموردين.
- مركز تسويات لإنشاء أوامر تسوية الموردين.
- تقارير تشغيل ومالية وتصدير CSV.
- إدارة العروض والكوبونات.
- مركز تنبيهات يعرض العناصر التي تحتاج إجراء.
- مركز مستندات لمراجعة اكتمال بيانات الموردين والسياسات.
- رقابة جاهزية النظام قبل التشغيل.
- دعم وتذاكر خدمة العملاء.
- إعدادات المنصة والاتصال والعملة والدولة والعمولات والضريبة داخل الإدارة فقط.
- تصدير واستيراد ملف احتياطي JSON.
- قوالب CSV فارغة للمنتجات والموردين والتغطية والدفعات والكوبونات والتسويات.
- ملف قاعدة بيانات Supabase داخل مجلد `supabase`.

## ملاحظات تشغيل

- لا توجد منتجات أو موردين أو طلبات أو حسابات جاهزة مسبقًا.
- العمولات والرسوم المالية والضريبة إعدادات داخلية للإدارة فقط.
- ظهور المنتجات في السوق يتطلب موردًا معتمدًا ومنتجًا منشورًا.
- قبول الطلب يتطلب وجود تغطية للمورد على المحافظة والمركز.
- البيانات تحفظ محليًا في المتصفح عند التشغيل المباشر. للبيئة الإنتاجية يتم ربط قاعدة بيانات فعلية مثل Supabase.
