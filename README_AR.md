# Tager Final Production Candidate V1

هذه نسخة نظيفة متقدمة مبنية على Next.js + Supabase + Vercel.
لا تعتمد على WordPress ولا ملفات Tager القديمة.

## أهم ما تم إضافته

- لا يوجد موردون أو منتجات تجريبية.
- صفحة إعداد أول مدير فقط: `/setup`.
- تسجيل العميل برقم الهاتف، والبريد اختياري.
- تسجيل المورد مع انتظار موافقة الإدارة.
- كل مورد يحدد أماكن التوصيل على مستوى:
  - المحافظة
  - المركز
  - القسم أو الحي
  - رسوم الشحن
  - مدة التوصيل
- العميل لا يستطيع إتمام الطلب إذا كان المورد لا يغطي مكان التوصيل.
- المراكز والأقسام تظهر حسب الموردين المعتمدين ومناطقهم فقط.
- إضافة منتجات بصور واضحة لا تقل عن 600×600.
- أسعار قطاعي / جملة / جملة الجملة.
- سلة منفصلة لكل مورد أو سلة مميزة برسوم المورد.
- طلبات وتتبع حالة الطلب.
- حساب مالي للمورد:
  - إجمالي المبيعات
  - عمولة المنصة
  - المدفوع
  - المتبقي
  - صافي المورد
- المورد يسجل دفعة عمولة وطريقة الدفع ورقم العملية.
- الإدارة تعتمد أو ترفض دفعات الموردين.
- قوائم مالية إدارية لكل مورد.
- فريق إدارة بصلاحيات.

## خطوات الرفع

1. فك الضغط.
2. ارفع محتويات المجلد إلى GitHub.
3. اربط GitHub مع Vercel.
4. في Vercel أضف:
   - NEXT_PUBLIC_SUPABASE_URL
   - NEXT_PUBLIC_SUPABASE_ANON_KEY
5. في Supabase شغل الملف:
   `supabase/schema_final.sql`
6. افتح الموقع على:
   `/setup`
7. أنشئ أول حساب إدارة حقيقي.

## مهم

هذه النسخة لا تحتوي بيانات موردين تجريبية.
ابدأ من حساب الإدارة الحقيقي، ثم سجّل الموردين والمنتجات الفعلية.

قبل الإطلاق التجاري الكبير يجب إضافة:
- بوابة دفع فعلية.
- SMS OTP إذا كان الدخول بالهاتف سيتم برسائل فعلية.
- سياسات RLS/Backend API للأمان العالي.
