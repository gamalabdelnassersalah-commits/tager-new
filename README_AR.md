# Tager Final Production V5 Complete

هذه نسخة شاملة ونظيفة مبنية على Next.js + Supabase + Vercel، بدون ملفات WordPress القديمة وبدون موردين أو منتجات تجريبية.

## الرفع

1. فك ضغط الملف.
2. افتح GitHub repo الحالي `tager-new`.
3. احذف كل الملفات القديمة.
4. ارفع محتويات هذا المجلد فقط، وليس ملف ZIP.
5. اعمل Commit.
6. افتح Supabase وشغّل `supabase/schema_final.sql` من SQL Editor.
7. تأكد أن Vercel يحتوي على:
   - `NEXT_PUBLIC_SUPABASE_URL`
   - `NEXT_PUBLIC_SUPABASE_ANON_KEY`
8. انتظر Deploy جديد.
9. افتح `/setup` وأنشئ أول حساب إدارة حقيقي.

## الصفحات المباشرة

- `/` الرئيسية
- `/setup` إعداد أول إدارة
- `/login` تسجيل الدخول
- `/register/customer` تسجيل عميل
- `/register/vendor` تسجيل مورد
- `/market` السوق والبحث
- `/vendors` دليل الموردين
- `/cart` السلة وإتمام الطلب
- `/customer` حساب العميل
- `/vendor` لوحة المورد
- `/admin` بوابة الإدارة
- `/how` كيف تعمل المنصة
- `/support` الدعم
- `/policies` السياسات

## الوظائف الأساسية

- تسجيل بالهاتف، والبريد اختياري.
- مورد تحت المراجعة حتى موافقة الإدارة.
- المنتج تحت المراجعة حتى موافقة الإدارة.
- المورد يحدد مناطق التوصيل حسب المحافظة، المركز، القسم/الحي.
- السلة العادية تمنع الطلب إذا كان المورد لا يغطي مكان العميل أو لم يتحقق الحد الأدنى.
- السلة المميزة تسمح بتجميع الموردين وتضيف نسبة مميزة لكل مورد.
- عمولة مختلفة لكل مورد.
- المورد يسجل دفعة عمولة، والإدارة تعتمد أو ترفض.
- قوائم مالية للمورد والإدارة.
- متابعة الطلبات والتوصيلات.
- صور المنتج والمورد لا تقل عن 600×600.

## ملاحظات مهمة قبل الإطلاق

- الدفع الحقيقي يحتاج ربط بوابة دفع حقيقية.
- SMS OTP يحتاج ربط مزود رسائل.
- راجع شروط الاستخدام وسياسة الخصوصية قانونيًا.
- بعد التشغيل الأولي أضف سياسات RLS في Supabase قبل فتح بيانات حساسة للجمهور.
