# Tager Clean MVP

نسخة جديدة نظيفة لا تعتمد على WordPress أو ملفات V5–V59.

## الرفع على GitHub ثم Vercel

1. فك ضغط الملف.
2. ارفع **كل الملفات الموجودة داخله** إلى مستودع GitHub جديد.
3. افتح Vercel واختر **Add New → Project**.
4. اختر مستودع GitHub.
5. اترك Framework Preset على **Other**.
6. لا تكتب Build Command ولا Output Directory.
7. اضغط **Deploy**.

سيظهر لك رابط يعمل مباشرة.

## حسابات الاختبار

- الإدارة: `01000000000` — كلمة المرور: `Admin@123`
- المورد: `01111111111` — كلمة المرور: `Vendor@123`
- العميل: `01222222222` — كلمة المرور: `Customer@123`

## ما يعمل في النسخة

- تسجيل العميل والمورد برقم الهاتف، والبريد اختياري.
- تسجيل الدخول والخروج وتوجيه كل نوع حساب إلى لوحته.
- مراجعة الموردين من الإدارة.
- إضافة المورد للمنتجات والصور والأسعار والمخزون.
- مراجعة المنتجات والموافقة عليها.
- صفحة مستقلة لكل مورد ومنتجاته.
- البحث حسب المنتج والمحافظة والمركز ونوع السعر.
- قطاعي وجملة وجملة الجملة.
- سلة الموردين المنفصلة.
- السلة المميزة برسوم قابلة للتعديل.
- عمولة مختلفة لكل مورد.
- الشحن المجاني بعد حد تحدده الإدارة.
- طرق دفع تجريبية.
- إنشاء الطلب ومتابعته من العميل والمورد والإدارة.
- إضافة موظفين للإدارة.
- عربي وإنجليزي وتصميم متجاوب.

## تنبيه مهم

هذه نسخة تجربة تعمل فورًا، وتحفظ البيانات داخل المتصفح باستخدام Local Storage. لذلك:

- البيانات التي تضيفها تظهر في نفس المتصفح والجهاز.
- فتح الرابط من جهاز آخر يبدأ بيانات تجريبية مستقلة.
- لا تستخدمها لتحصيل أموال حقيقية أو حفظ بيانات عملاء حقيقية.
- بعد اعتماد التصميم والوظائف، يتم ربطها بقاعدة بيانات Supabase لتصبح نسخة إنتاج مشتركة بين جميع المستخدمين.
