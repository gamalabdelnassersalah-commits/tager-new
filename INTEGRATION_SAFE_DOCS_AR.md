# تقرير دمج الفاتورة وطلب الشراء مع منصة Tager

تم الدمج بطريقة آمنة بحيث تعمل ملفات المنصة وملفات مستندات Tager معًا بدون استبدال ملفات المنصة الأساسية.

## ما تم عمله
- إضافة مجلد `tager-documents` إلى جذر المشروع وإلى `public`.
- ربط صفحات الطباعة داخل المنصة بالمسارات التالية:
  - `vendorInvoicePrint` -> `tager-documents/invoice.html`
  - `invoicePrint` -> `tager-documents/invoice.html`
  - `orderPrint` -> `tager-documents/purchase-order.html`
- تمرير بيانات الطلب الحقيقية إلى القالب عبر `data` في الرابط.
- عرض المستند داخل المنصة داخل iframe، مع زر فتح مستقل وزر طباعة / حفظ PDF.
- تعديل `build-static.mjs` لنسخ مجلد `tager-documents` عند البناء.

## ملاحظات مهمة
- لم يتم تغيير جداول أو بيانات العملاء أو الموردين أو الطلبات.
- لم يتم استبدال المنصة بملفات المستندات.
- تم الحفاظ على فصل CSS/JS الخاص بالمستندات داخل نطاق مستقل حتى لا يؤثر على صفحات المنصة.

## اختبار الرفع
1. ارفع محتوى هذا الملف على GitHub.
2. على Vercel سيعمل الأمر `npm run build`.
3. بعد النشر اختبر:
   - طلب الشراء من زر طباعة الطلب.
   - فاتورة المورد من زر طباعة الفاتورة في لوحة المورد.
   - فاتورة العميل من صفحة تتبع الطلب.
