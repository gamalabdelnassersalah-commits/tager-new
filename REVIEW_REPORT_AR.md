# تقرير مراجعة الحزمة السابقة

تمت مراجعة ملفات الحزمة السابقة الخاصة بفاتورة المورد وطلب الشراء.

## المشاكل التي كانت ممكن تدمّر المنصة

1. وجود `index.html` في جذر الحزمة، وهذا قد يستبدل الصفحة الرئيسية إذا تم رفعه مكان ملفات المنصة.
2. ملف CSS كان يحتوي على قواعد عامة مثل `html`, `body`, `*`، وهذه القواعد ممكن تغيّر تصميم المنصة كلها إذا تم استيرادها داخل React/Next.
3. أسماء CSS كانت عامة مثل `info-card`, `items-table`, `doc-actions`، وقد تتداخل مع كروت وجداول المنصة.
4. JavaScript كان يبحث داخل `document` بالكامل، وليس داخل نطاق محدد، وهذا ممكن يؤثر على عناصر أخرى داخل المنصة.
5. لو تم نسخ ملفات HTML بدون مجلد `assets` ستظهر الصفحة بيضاء أو بدون صور وأيقونات.
6. الملفات كانت مناسبة للعرض المستقل، وليست دمج مباشر داخل منصة Next.js.

## الإصلاح في هذه النسخة

1. كل التصميم معزول تحت `.tgr-doc-scope`.
2. تم حذف أي CSS عام يؤثر على جسم المنصة.
3. تم عزل JavaScript ليعمل فقط داخل قالب Tager.
4. تم وضع الملفات داخل `public/tager-documents` حتى يتم ربطها كرابط طباعة مستقل.
5. تم إضافة تكامل آمن مع Next.js باستخدام iframe أو رابط فتح صفحة الطباعة.
6. لا يوجد ملف `index.html` في جذر الحزمة.

## طريقة الاستخدام الآمنة

انسخ فقط:

```txt
public/tager-documents
```

إلى:

```txt
platform/public/tager-documents
```

ثم اربط زر الفاتورة أو طلب الشراء بالرابط المناسب.
